How to use

Step 1 — Export your network in-game

Install the AE2 Network Analyser mod, then run this command while connected to your ME network:

/ae2 grids export

↑ Click the command to copy it to your clipboard.

This writes a .zip file to your Minecraft instance directory (next to options.txt). The archive contains one grid_*.json per sub-network and a chunks/ folder with block-entity SNBT data.

⚠ Requires cheats / operator permissions
Singleplayer: If your world was not created with cheats enabled, open it to LAN (Esc → Open to LAN → Allow Cheats: ON → Start LAN World) before running the command.

Multiplayer / server: Ask your server owner to grant you /ae2 command privileges (operator level or a permission plugin entry for ae2.command.grids).
Step 2 — Load the file

Click Load files in the Data Source panel and select the .zip. You can also select individual grid_*.json or .snbt files directly if you have the archive unpacked.

All grids and dimensions are loaded at once. Use the 3D Dimension selector to switch which world is shown in the viewport.

The camera auto-fits to the loaded data when you change the dimension or reload a file.

Filters
LevelMatch dimension ID. Supports * wildcards — e.g. *overworld* or allthemodium:*
ItemMatch item ID. Wildcards supported — e.g. ae2:*cable* or *storage_bus*
X / Y / ZMin and/or max coordinate bounds. Leave either blank to apply only the other bound.
FlagsShow only items with a given flag (SUS, STARVED, CAP, VIRT, CONS) or unflagged items.
BlacklistEnter mod:item or mod:* to hide an item or an entire mod. Blacklisted items are excluded from demand calculations.
Non-consumersItems that look like consumers but should not use channels. Excluded from demand calculations.

When a filter is active and Show non-matching as ghosts is on, non-matching blocks remain visible in the 3D view at low opacity for context.

Table interactions
Left-click rowCopy /tp @s X Y Z to clipboard — paste it in-game to teleport straight to the block.
Right-click rowOpen context menu: fly the 3D camera to that block, filter the table for that item, or copy the coordinates.
Column headerSort ascending / descending by that column.
Flags explained
SUS A cable bus whose reachable consumer chain exceeds the cable's channel capacity. The whole chain is walked — consumers adjacent to other consumers (passthrough) are all counted.
STARVED A consumer on a network island where total demand exceeds the island's available capacity (cables + virtual links, controller-aware).
CAP A block that provides channel capacity — smart cables (8 ch), dense cables (32 ch), or a controller.
VIRT A virtual channel link: P2P tunnel, wireless access point, quantum link / ring / bridge.
CONS A device that consumes at least one channel. Multiblock structures (assembler matrix, quantum computer) count as one consumer regardless of size.
Tips
  • Adjust the Cable Config thresholds if you use mods that change channel limits — all flags recalculate instantly.
  • Use the SNBT Whitelist to restrict the view to specific mod namespaces if your network mixes many mods.
  • Add ?debug=1 to the URL and reload to print a detailed parse summary to the browser console.
  • The export contains no live channel counts — SUS and STARVED detection is heuristic, based on cable type and physical topology.
